# Рекомендации по UX и производительности (фокус на мобильные)

## Что проверено
- Просмотрен код структуры страницы и загрузки зависимостей (`index.html`, `js/app.js`, `js/vendor-loader.js`, `js/map-init.js`, `css/style.css`).
- Запущен быстрый мобильный прогон страницы (390x844, touch):
  - `domContentLoaded ≈ 677ms`
  - `loadEventEnd ≈ 2000ms`
  - `FCP ≈ 1188ms`

## Критичные улучшения (сначала)

1. **Убрать `backdrop-filter: blur(...)` на мобильных или сильно снизить blur**  
   Сейчас blur применяется к нескольким плавающим панелям (`.map-banner`, `.map-stats`, `.map-reset-button`, `.map-error`) и может нагружать GPU на средних Android-устройствах.  
   **Что сделать:** для `max-width: 720px` заменить на полупрозрачный фон без blur, или оставить blur 4–6px вместо 18px.

2. **Снижать стоимость карты на слабых устройствах не только по ширине, но и по сети/CPU**  
   Сейчас 3D отключается в основном при узком viewport и `prefers-reduced-motion`. Для части телефонов в портретном режиме этого недостаточно.  
   **Что сделать:** добавить эвристику `navigator.hardwareConcurrency`, `navigator.deviceMemory` (если доступно), `navigator.connection?.effectiveType`, и при слабых параметрах:
   - отключать terrain/fog,
   - уменьшать анимации (`duration` для `fitBounds/easeTo`),
   - уменьшать количество интеракций в первый экран.

3. **Ленивая инициализация не-критичных UI блоков**  
   Визуально важен только заголовок + карта. Достижения и расширенная статистика можно отложить до `requestIdleCallback`/первого взаимодействия.  
   **Что сделать:** рендер `renderAchievements`/части `renderStats` перенести в idle-фазу.

4. **Добавить деградацию для флагов в списках**  
   Сейчас иконки флагов грузятся с CDN. На мобильной сети это даёт дополнительные DNS/TLS/HTTP-запросы и задержки открытия дропдаунов.  
   **Что сделать:**
   - по умолчанию на мобильных использовать emoji-флаги,
   - либо подгружать изображения флагов только при открытии конкретного дропдауна.

## UX-улучшения для мобильного сценария

5. **Улучшить «безошибочный тап» по элементам управления**  
   Кнопка info и элементы статистики местами мелковаты, особенно при масштабировании интерфейса браузером.  
   **Что сделать:** обеспечить минимальный touch-target 44x44px для всех кликабельных контролов.

6. **Упростить первый экран**  
   На мобильном одновременно видны banner + achievements + stats + reset. Это перегружает карту и отвлекает от ключевого сценария (тап по маркеру).  
   **Что сделать:**
   - на первом экране оставить banner + reset,
   - stats свернуть в компактный чип «Статистика» с bottom-sheet,
   - achievements скрывать за «🏆» кнопкой.

7. **Ясные состояния загрузки**  
   При медленной сети карта и данные появляются асинхронно; пользователь может видеть «пустой» интерфейс.  
   **Что сделать:** добавить скелетон/лейбл «Загружаем карту и точки…» + индикатор статуса загрузки таблицы.

8. **Проверить жесты и блокировку прокрутки страницы**  
   Для map-приложений на iOS/Android критично, чтобы жесты карты не конфликтовали с системным скроллом/overscroll.  
   **Что сделать:** добавить и протестировать `touch-action`/`overscroll-behavior` стратегию для контейнера карты.

## Сетевые и runtime оптимизации

9. **Профилировать и ограничить пересчёты при resize/rotate**  
   Сейчас есть debounce 150ms и вызовы `refresh3DLayers` + `resize`. При повороте телефона бывает серия resize-событий.  
   **Что сделать:**
   - увеличить debounce до 250ms для мобильных,
   - при rotate запускать один «финальный» перерасчёт.

10. **Кэширование данных Google Sheets**  
   Сейчас данные тянутся при каждом открытии.  
   **Что сделать:** хранить нормализованные данные в `localStorage` с TTL (например 12–24 часа), показывать кэш мгновенно, затем тихо обновлять в фоне.

11. **Шрифты**  
   Google Fonts уже подключены корректно, но можно ещё снизить риск FOIT/FOUT на слабой сети.  
   **Что сделать:**
   - оставить только реально используемые веса,
   - добавить локальный системный fallback-приоритет при медленной сети.

## Что внедрять по этапам

### Этап 1 (быстрые победы, 1–2 дня)
- Отключить/ослабить blur на мобильных.
- Включить emoji-флаги по умолчанию на мобильных.
- Добавить крупные touch-target (44x44).
- Добавить явный loading-state карты/данных.

### Этап 2 (2–4 дня)
- Ввести адаптивный «low-power mode» (по устройству/сети).
- Отложить achievements/stats в idle.
- Переработать мобильную компоновку в сторону bottom-sheet.

### Этап 3 (4–7 дней)
- Кэш данных Google Sheets с TTL и фоновым refresh.
- Тонкая настройка resize/rotate-пайплайна.
- Сравнительный замер Web Vitals (mobile) до/после.

## Целевые KPI для мобильной версии
- **FCP < 1.8s** на 4G mid-tier Android.
- **LCP < 2.5s** на 4G mid-tier Android.
- **INP < 200ms** при взаимодействии с маркерами/дропдаунами.
- **Стабильные 50–60 FPS** при pan/zoom на карте без 3D-эффектов.
